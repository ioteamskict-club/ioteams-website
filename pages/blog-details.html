Getting Started with MQTT Protocol for IoT Devices — A Complete Guide

MQTT is the backbone of most IoT communication systems. In this guide, we walk you through setting up a broker, publishing and subscribing to topics, and connecting your ESP32 to the cloud — step by step with code examples.

Ahmad Faris
President · IoTeams IIUM
Share:
blog-featured-cover.jpg

If you've been working with IoT devices, you've almost certainly heard of MQTT. It's lightweight, efficient, and designed specifically for constrained networks and devices — which makes it perfect for the ESP32, Arduino, and Raspberry Pi projects we build at IoTeams.

In this tutorial, we'll cover everything from the basic concepts of MQTT to a complete working example sending sensor data from your ESP32 to a cloud dashboard. By the end, you'll have a broker running, devices publishing data, and a subscriber reading it all in real time.

Prerequisites: Basic Arduino/ESP32 programming knowledge. You should be comfortable uploading a sketch and reading serial output. No cloud experience required.

What is MQTT?

MQTT stands for Message Queuing Telemetry Transport. It's a publish-subscribe messaging protocol originally designed by IBM in the late 1990s for monitoring oil pipelines via satellite. Today it's the de facto standard for IoT communication.

The protocol has three key components — the broker, publishers, and subscribers.

  • The broker is a central server that receives all messages and routes them to the right subscribers.
  • Publishers are devices (like your ESP32) that send data to the broker.
  • Subscribers are clients that listen for messages on specific topics.

"MQTT is to IoT what HTTP is to the web. Once you understand it, you can build almost any connected system."

Setting Up a Mosquitto Broker

The most popular open-source MQTT broker is Mosquitto. We'll install it locally first, then discuss cloud options.

Installing Mosquitto

On Ubuntu/Debian (including Raspberry Pi OS):

bash 
sudo apt update
sudo apt install -y mosquitto mosquitto-clients
sudo systemctl enable mosquitto
sudo systemctl start mosquitto

Verify it's running:

bash 
sudo systemctl status mosquitto
# Should show: active (running)

By default, Mosquitto only accepts connections from localhost. To allow external devices, you'll need to edit /etc/mosquitto/mosquitto.conf and add listener 1883 and allow_anonymous true. Don't do this on a public network without authentication.

Understanding Topics

MQTT uses a topic-based routing system. Topics are hierarchical strings separated by slashes, similar to file paths:

topics 
ioteams/sensors/temperature
ioteams/sensors/humidity
ioteams/lab/door/status
ioteams/actuators/fan/speed

Subscribers can use wildcards: + matches a single level, # matches all remaining levels.

Connecting ESP32 via MQTT

We'll use the PubSubClient library for Arduino. Install it via the Arduino Library Manager.

C++ (Arduino) 
#include <WiFi.h>
#include <PubSubClient.h>
#include <DHT.h>

// WiFi credentials
const char* ssid     = "YOUR_WIFI_SSID";
const char* password = "YOUR_WIFI_PASSWORD";

// MQTT broker (your Raspberry Pi or cloud IP)
const char* mqtt_server = "192.168.1.100";
const int   mqtt_port   = 1883;

// Topics
const char* topic_temp = "ioteams/sensors/temperature";
const char* topic_hum  = "ioteams/sensors/humidity";

// DHT22 sensor on pin 4
DHT dht(4, DHT22);
WiFiClient espClient;
PubSubClient client(espClient);

void connectWiFi() {
  WiFi.begin(ssid, password);
  while (WiFi.status() != WL_CONNECTED) {
    delay(500);
    Serial.print(".");
  }
  Serial.println("\n✓ WiFi connected: " + WiFi.localIP().toString());
}

void connectMQTT() {
  while (!client.connected()) {
    String clientId = "ESP32-" + String(random(0xffff), HEX);
    if (client.connect(clientId.c_str())) {
      Serial.println("✓ MQTT connected");
    } else {
      Serial.print("MQTT failed, rc=");
      Serial.println(client.state());
      delay(2000);
    }
  }
}

void setup() {
  Serial.begin(115200);
  dht.begin();
  connectWiFi();
  client.setServer(mqtt_server, mqtt_port);
  connectMQTT();
}

void loop() {
  if (!client.connected()) connectMQTT();
  client.loop();

  float temp = dht.readTemperature();
  float hum  = dht.readHumidity();

  if (!isnan(temp) && !isnan(hum)) {
    client.publish(topic_temp, String(temp).c_str());
    client.publish(topic_hum,  String(hum).c_str());
    Serial.printf("Published: temp=%.1f°C hum=%.1f%%\n", temp, hum);
  }

  delay(5000); // Publish every 5 seconds
}

Tip: In production, never hardcode credentials. Use EEPROM or a config file approach, or flash credentials separately using WiFiManager.

Testing Your Setup

With Mosquitto installed and your ESP32 running, open two terminal windows. In one, subscribe to the temperature topic:

bash 
mosquitto_sub -h localhost -t "ioteams/sensors/#" -v

You should see output like:

output 
ioteams/sensors/temperature 27.3
ioteams/sensors/humidity 65.2
ioteams/sensors/temperature 27.4
ioteams/sensors/humidity 65.1

Next Steps

Now that you have basic MQTT working, here's what to explore next:

  1. Add authentication — configure Mosquitto with username/password
  2. Enable TLS — encrypt your MQTT traffic with SSL certificates
  3. Use Node-RED — visually wire your MQTT data to dashboards and alerts
  4. Try AWS IoT Core — scale to thousands of devices with managed cloud MQTT
  5. Implement QoS — understand Quality of Service levels 0, 1, and 2
